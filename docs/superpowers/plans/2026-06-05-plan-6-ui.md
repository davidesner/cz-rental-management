# Plan 6 — UI

**Goal:** Vite + React + Tailwind + shadcn/ui web UI covering auth, dashboard, properties, tenants, contracts (with terms/utilities), tariffs, payments, cost statements, reconciliations. Chrome MCP smoke tests for critical paths (register → org → property → reconciliation).

## Scope

MVP UI — functional over fancy. Each page is a table + a "Create / Edit" dialog. No fancy interactions.

## Stack

- Vite (already in package.json scripts? — no, we need to add)
- React + react-router-dom v7
- Tailwind CSS + shadcn/ui (radix-based components)
- @tanstack/react-query for fetching
- react-hook-form + Zod for forms
- Vitest + @testing-library/react for component tests
- Chrome DevTools MCP for E2E smoke tests

## Pages

- `/login`, `/register`
- `/` — dashboard: orgs, current org, properties summary, recent reconciliations
- `/properties`, `/properties/:id` (with sub-sections for tariff history)
- `/tenants`, `/tenants/:id`
- `/contracts`, `/contracts/:id` (with sub-sections for terms history, utilities)
- `/payments` — table with inbox filter + assign action
- `/cost-statements` — table with filters
- `/reconciliations`, `/reconciliations/:id` — create + view
- `/settings/api-tokens` — manage tokens for MCP

## Tasks

1. Vite + Tailwind + shadcn setup; React entry + router shell.
2. Auth pages + session hook.
3. App layout (sidebar nav, org switcher, sign-out).
4. Dashboard.
5. Resource pages: properties, tenants, contracts (incl. nested terms/utility tabs), tariffs, payments, cost statements, reconciliations, api tokens.
6. Chrome MCP smoke test script: register → create org → create property → view reconciliation result.
